TrustSurface Framework

TrustSurface is a neutral, evidence-led framework for identifying, measuring, and governing the observable trust signals an organisation emits at its digital edge.

It exists because organisations can be internally well-controlled and still appear untrustworthy through weak signals: spoofable email, brittle DNS, inconsistent identity boundaries, unreliable services, and unmanaged vendor integrations.

What it makes governable

  • Trust Surface — the externally-facing systems where trust is experienced.
  • Trust Signals — observable indicators (evidence) describing posture.
  • Trust Posture — the evidence-based state implied by signals.
  • Trust Signal Gap — intended posture ≠ observed evidence.

Core definitions are stabilised in Glossary v1.0.

The visual anchor

Trust Surface domains

  • Identity boundary — how people and services prove who/what they are.
  • Domains & DNS — ownership, routing, integrity, and availability.
  • Email integrity — whether mail claiming your domain can be trusted.
  • Digital services — websites, portals, APIs, and public application edge.
  • Infrastructure & platforms — the hosting and platform layer underpinning the edge.
  • Third‑party ecosystem — delegated trust through vendors and SaaS.

Operating rhythm

Discover → Assess → Harden → Govern → Signal is the minimal cycle that converts trust into evidence, prioritised work, and governance controls.

How to use it (minimal)

  1. Read the one-page specification for a concise overview.
  2. Use the adoption guidance to set cadence, roles, and evidence artefacts.
  3. Use the worked example to see the method applied end-to-end.
  4. Use the library for the full document set and stable links.

What this is not

TrustSurface is a lens, not a control framework. It complements ISO 27001, NIST CSF, and COBIT by translating internal programs into externally meaningful evidence. See comparative positioning.
